Better control over properties

  • In ES3, propeties were name/value pairs. In ES5, properties have "writable", "enumerable", "configurable" attributes, default value being 'false'.
  • "writable" attribute controls assignment of values, "enumerable" hides a property during inspection of object, "configurable" controls alteration/deletion of properties.
  • E.g., If "writable" attribute is false, new values cannot be assigned.
